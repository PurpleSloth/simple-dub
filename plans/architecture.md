# Архитектура Simple Dub

## Цель

Одноголосый русский автодубляж без чтения субтитров. Приложение не заменяет
исходные потоки: оно добавляет в новый MKV отдельную аудиодорожку с русским
голосом поверх приглушённого оригинала.

## Маршруты

```text
русские текстовые субтитры ───────────────┐
иностранные текстовые субтитры → перевод ├→ Silero v5_5_ru
нет текста / PGS / VobSub → whisper.cpp → перевод ┘
                                               ↓
 выбранная stereo-дорожка → приглушение → mix → новый MKV
```

- ASR получает отдельную копию выбранной дорожки: mono, 16 кГц. Это улучшает
  распознавание и не влияет на финальный звук.
- Для микса исходная выбранная дорожка остаётся stereo.
- Все исходные видео, аудио, субтитры, главы, вложения и метаданные копируются.
- Новый AAC-трек получает язык `rus` и название
  `Русский одноголосый дубляж`.

## Границы компонентов

- `src/`: интерфейс выбора файла и потоков.
- `src-tauri/`: безопасный запуск локальных процессов и desktop API.
- `core/`: чистые решения маршрутизации, команды, разбор потоков и кэш.
- `runtime/`: скачанные бинарники и модели; не хранится в Git.
- ключ OpenRouter: Windows Credential Manager; frontend видит только статус.

Официальный `v5_5_ru.pt` является `torch.package`. Проверенный Python-worker
хранится локально в `_legacy_python/` и исключён из Git. Production-интеграция
TTS должна получить отдельный упакованный sidecar либо другую нативную модель;
до этого TTS не считается подключённым к desktop-сборке.

## Модели

- ASR: `whisper.cpp` + `ggml-large-v3-turbo`, CUDA, VAD.
- Перевод: `google/gemini-3.5-flash-lite` через OpenRouter.
- TTS: Silero `v5_5_ru`.

Qwen и Python-реализация Whisper в новом пайплайне не используются.
