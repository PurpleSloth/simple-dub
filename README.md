# Simple Dub

Desktop-приложение для минимального одноголосого дубляжа видео на русский язык.
Приложение сохраняет исходные потоки Matroska и добавляет новую аудиодорожку
с озвучкой поверх приглушённого оригинала.

## Целевой сценарий

1. Пользователь открывает видео.
2. Приложение показывает аудиодорожки и субтитры из контейнера.
3. Пользователь выбирает оригинальную аудиодорожку.
4. Если выбраны русские текстовые субтитры, они сразу отправляются в TTS.
5. Иностранные текстовые субтитры переводятся через OpenRouter.
6. Если подходящих субтитров нет, речь распознаётся локально через `whisper.cpp`.
7. Выбранный TTS-движок создаёт одноголосую русскую озвучку.
8. `ffmpeg` собирает новый MKV, сохраняя исходные видео, аудио, субтитры,
   главы, вложения и метаданные.

Графические PGS/VobSub-субтитры в первой версии не распознаются через OCR:
для них используется ветка `whisper.cpp`.

## Стек

- Tauri 2 и Rust — desktop-интерфейс и оркестрация;
- TypeScript и Vite — интерфейс;
- `ffprobe`/`ffmpeg` — анализ, микширование и Matroska mux;
- `whisper.cpp` `large-v3-turbo` — локальное распознавание речи;
- Piper Dmitri FP32 через `sherpa-onnx` — основной нативный голос;
- Silero `v5_5_ru`, Eugene — дополнительный голос в автономном CPU-worker;
- OpenRouter `google/gemini-3.5-flash-lite` — перевод субтитров.

Qwen в новой архитектуре не используется.

Ключ OpenRouter вводится в настройках установленного приложения и хранится
в Windows Credential Manager. Tauri API возвращает интерфейсу только признак
наличия ключа, но никогда не его значение.

## Что уже работает

Приложение:

- открыть видео через системный диалог;
- разобрать JSON `ffprobe`;
- показать доступные аудио- и субтитровые потоки;
- отличить текстовые субтитры от PGS/VobSub;
- выбрать маршрут: озвучка, перевод или `whisper.cpp`;
- выбрать Piper Dmitri FP32 или Silero Eugene для конкретного задания;
- запомнить последний выбранный TTS-движок;
- извлечь выбранные текстовые субтитры или распознать выбранную аудиодорожку;
- перевести иностранные реплики с учётом доступной длительности;
- синтезировать и выровнять каждую реплику по исходным таймкодам;
- смешать голос с приглушённым, но сохранённым в stereo оригиналом;
- создать рядом с исходником новый `.dub.ru.mkv`, не перезаписывая готовые
  результаты;
- показывать ход задания и итоговый путь;
- сохранить все исходные потоки, главы, вложения и метаданные и добавить
  отдельную русскую AAC stereo-дорожку.

Маршруты с русскими субтитрами, иностранными субтитрами и локальным
распознаванием проверены реальными короткими MKV. Оба TTS-варианта также
проверены полным запуском.

## Установка для пользователя

Установите `Simple Dub_0.1.0_x64-setup.exe`. FFmpeg и ffprobe входят в
установщик. При первом задании приложение автоматически скачает только
необходимые модели:

- Piper Dmitri FP32 — если выбран Piper;
- автономный CPU-worker и модель — если выбран Silero Eugene;
- CPU-сборку `whisper.cpp`, `large-v3-turbo` и VAD — только если нужно
  распознавать речь.

Системный Python, FFmpeg и NVIDIA GPU не требуются. Для первого скачивания
компонентов нужен интернет. Для перевода нужен ключ OpenRouter: он вводится
через «Настройки» и сохраняется в Windows Credential Manager.

## Разработка

Требуются Node.js 20+, Rust stable и Windows 10/11 с WebView2.

Установка frontend-зависимостей:

```powershell
npm install
```

Локальная подготовка выбранных компонентов:

```powershell
pwsh -NoProfile -File scripts/setup-runtime.ps1 `
  -RuntimeRoot runtime `
  -InstallPiper `
  -InstallWhisper `
  -PiperWorkerSource src-tauri/bin/piper-worker-x86_64-pc-windows-msvc.exe
```

Бинарники и модели сохраняются в `runtime/` и не попадают в Git.

Запуск приложения в режиме разработки:

```powershell
npm run tauri dev
```

## Проверки

Rust-тесты, включая создание и повторный анализ временного MKV:

```powershell
cargo test -p simple-dub-core
```

Проверка Tauri:

```powershell
cargo check -p simple-dub-desktop
```

Production-сборка интерфейса:

```powershell
npm run build
```

Сборка NSIS-инсталлятора:

```powershell
npm run desktop:build
```

Воспроизводимая сборка автономного Silero-worker из нового Python-исходника:

```powershell
pwsh -NoProfile -File scripts/build-silero-worker.ps1
```

Скрипт использует CPython 3.12, создаёт отдельную среду в игнорируемом
`output/`, устанавливает закреплённые CPU PyTorch, NumPy и PyInstaller и
печатает SHA-256 готового `silero-worker.exe`.

Проверка файлов будущего коммита на ключи OpenRouter:

```powershell
npm run check:secrets
```

## Структура

```text
core/                 Чистое Rust-ядро и тесты
src/                  TypeScript-интерфейс
src-tauri/            Tauri desktop-приложение
scripts/              Установка локального runtime
runtime/              Бинарники и модели, игнорируется Git
native/                Нативный worker Piper
workers/               Новый исходник и зависимости Silero-worker
_legacy_python/        Локальный Python-код вне Git
```

Старый Python-проект хранится локально в `_legacy_python/` и не попадает в
Git. Новый минимальный Silero-worker и его закреплённые build-зависимости
лежат в `workers/`, поэтому release-бинарник можно восстановить из репозитория.
Скомпилированный worker скачивается приложением только при выборе Eugene.

## Лицензии моделей

`whisper.cpp` и модели Whisper распространяются отдельно на условиях их
лицензий. Большинство публичных моделей Silero, включая русские TTS-модели,
имеют некоммерческую лицензию CC-NC-BY. Перед коммерческим распространением
нужно отдельно проверить условия использования.
